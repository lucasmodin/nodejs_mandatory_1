Git basics

Git is a open source version control system. It allows developers to work on different feature, track changes, and collaborate efficiently.

Below are some common git commands that are useful for every project.

Get started

Every Git project starts by creating or cloning a repository. A repository is a folder that Git tracks changes in. 

git init
git clone <url>

Prepare to commit

Before saving your changes you need to tell Git which files to include. This is done by using the staging area. 

git add <file>
git add .
git rm <file>
git reset <file>
git reset

Make commits

A commit is like a snapshot of your code. It stores all staged changes with a short message describing what work was done. 

git commit
git commit -m 'message'
git commit -am 'message'

Branch

Branches let you work on new features or fixes without changing the main code (usually called main or master). 

git branch
git branch <name>
git checkout <name>
git checkout -b <name>
git merge <name>

Use git branch to list branches, git checkout to switch, and git merge to combine changes.

Push

After committing your changes, you can upload them to a remote repository like GitHub using push. 

git push
git push origin <branch>
