Git basics

Git is an open-source version control system. It allows developers to work on different features, track changes, and collaborate efficiently.

Below are some common Git commands that are useful for every project.

Get started

Every Git project starts by creating or cloning a repository. A repository is a folder that Git tracks changes in. 

git init

Initializes a new Git repository

git clone <url>

Copies an existing remote repository

Prepare to commit

Before saving your changes you need to tell Git which files to include. This is done using the staging area. 

git add <file>

Stages a specific file for the next commit

git add .

Stages all modified files in the project

git rm <file>

Removes a file from both your working directory and Git history

git reset <file>

Unstages a specific file from the staging area

git reset

Unstages all files currently staged for commit

Make commits

A commit is like a snapshot of your code. It stores all staged changes with a short message describing what was done. 

git commit

Opens an editor to write a commit message

git commit -m "message"

Creates a commit with a short message

git commit -am "message"

Stages and commits all tracked files

Branch

Branches let you work on new features or fixes without changing the main code (usually called main or master). 

git branch

Lists all existing branches

git branch <name>

Creates a new branch

git checkout <name>

Switches to an existing branch

git checkout -b <name>

Creates and switches to a new branch

git merge <name>

Merges another branch into the current one

Use git branch to list branches, git checkout to switch, and git merge to combine changes.

Push

After committing your changes, you can upload them to a remote repository using push. 

git push

Uploads your commits to the remote repository

git push origin <branch>

Pushes a specific branch to the remote repository
